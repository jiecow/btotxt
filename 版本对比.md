## 一、需求提出

用户描述 BiliNote 的核心技术链路：

> BiliNote 把 B 站视频链接转为文本，核心技术链路是：视频解析下载 → 音频提取 → ASR 语音识别，具体如下：
> 1. **视频下载解析**：用 yt-dlp（或 you-get）解析 B 站链接，优先尝试抓取已有弹幕/CC 字幕；若无字幕则下载音视频流（可只下音频）。
> 2. **音频提取**：用 FFmpeg 从视频容器中分离出音频轨道（如转成 16kHz 单声道 WAV/MP3），为 ASR 模型准备标准输入。
> 3. **语音转文字（ASR）**：BiliNote 支持多种转写引擎，核心是 OpenAI Whisper：本地转写（faster-whisper）、云端 API（Groq Whisper API、必剪 ASR 等）。
> 4. **后续处理**：转写文本再交给 GPT / DeepSeek / Qwen 等大语言模型做摘要、分段、生成带时间戳的 Markdown 笔记。
>
> 请帮我设计并简化这套流程，要求部署到本地服务器上，通过 http://localhost:3015 进行访问。后续处理使用 deepseek API。

## 二、设计方案

### 环境调研

- **目录**：`/home/jiecow/test/bilisum/` — 已有 `BiliSum.txt` 笔记（Docker 方案 lycohana/bilisum）
- **系统**：Linux x86_64, CPU only（无 GPU）, 8GB RAM
- **已装**：ffmpeg, Python 3.14, Node 22
- **未装**：yt-dlp, faster-whisper 等

### 关键决策

1. **ASR 方案**：用户选择「纯本地 faster-whisper CPU 模式」— 完全免费但速度慢
2. **Web 框架**：Python Flask + HTML 模板（最轻量）
3. **LLM 后处理**：DeepSeek API
4. **端口**：3015

### 架构设计

```
用户浏览器 ──HTTP──→ Flask Web App ──→ yt-dlp (下载音频)
                                        → FFmpeg (转 16kHz WAV)
                                        → faster-whisper (CPU ASR)
                                        → DeepSeek API (摘要/笔记)
                                        → 结果渲染到页面
```

### 页面流程

1. **首页**：粘贴 B 站链接 + 提交
2. **处理中页面**：实时进度轮询（每 2 秒）
3. **结果页**：AI 笔记 → 整理后文本 → 带时间戳转写

## 五、增加 10 种笔记风格

### 需求

用户要求在默认风格之外增加 10 种可选笔记风格：
详细、精简、教程、学术、小红书、生活向导、任务导向、商业风格、会议纪要、中英文对照

### 实现方式

在 `llm_handler.py` 中定义 `NOTE_STYLES` 字典，每种风格包含 `label` 和 `system`（system prompt）。

### 修改的文件

| 文件 | 改动 |
|---|---|
| `llm_handler.py` | 新增 `NOTE_STYLES` 字典（10 种风格）；`generate_notes()` 接受 `style` 参数 |
| `task_manager.py` | `create_task()` 增加 `note_style` 参数；流水线传递风格 |
| `app.py` | 首页传入 `note_styles`；`/submit` 读取 `note_style` |
| `index.html` | 新增风格选择器（胶囊按钮样式，高亮交互） |
| `result.html` | 结果页显示当前笔记风格标签 |
| `style.css` | 新增 `.style-selector`、`.style-option`、`.style-badge` 样式 |
